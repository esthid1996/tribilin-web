🚀 Despliegue en GitHub Pages - Tribilin

Guía paso a paso para publicar tu sitio en internet
✅ ¡Sitio listo para publicar!

Tu página web de Tribilin está completamente lista. Solo necesitas 5 minutos para ponerla en línea.

Parte 1: Preparación (1 minuto)

1
Descarga Git para Windows

Si no tienes Git instalado:

  1. Ve a: https://git-scm.com/download/win
  2. Descarga e instala (acepta todos los valores predeterminados)
  3. Reinicia tu computadora después de instalar
💡 Nota: Si ya tienes Git instalado, puedes saltar este paso.
2
Crea una cuenta en GitHub (Si no la tienes)
  1. Ve a: https://github.com/signup
  2. Ingresa tu email, contraseña y usuario (ej: tu_nombre_usuario)
  3. Completa la verificación
💡 Nombre de usuario recomendado: algo simple como "tribilin-web" o "esthid-raul"

Parte 2: Crear Repositorio en GitHub (2 minutos)

3
Crea un nuevo repositorio
  1. Inicia sesión en GitHub: https://github.com/login
  2. En la esquina superior derecha, haz clic en +
  3. Selecciona New repository
  4. Completa el formulario:
    Repository name: tribilin-web Description: Página web de Tribilin - Servicios TI Public: ☑ (Debe estar público para Pages) Add README.md: ☐ (No necesario)
  5. Haz clic en Create repository
💡 Importante: El repositorio DEBE ser público para que GitHub Pages funcione en la versión gratuita.

Parte 3: Subir Archivos (2 minutos)

4
Opción A: Subir desde GitHub (Más Fácil)
  1. En la página del repositorio, haz clic en Add file → Upload files
  2. Arrastra tu carpeta del proyecto o haz clic para seleccionar archivos
  3. Sube todos los archivos de la carpeta Claude_proyecto:
    • ✓ index.html
    • ✓ informe_archivos.html
    • ✓ README.md
    • ✓ css/styles.css
    • ✓ js/script.js
    • Otros archivos (.txt, .html, etc)
  4. En el campo "Commit message", escribe: Primer despliegue de Tribilin
  5. Haz clic en Commit changes
✅ Los archivos se cargarán automáticamente
⚠️ IMPORTANTE: Asegúrate de que index.html esté en la raíz del repositorio, no dentro de una carpeta.

Parte 4: Habilitar GitHub Pages (1 minuto)

5
Activar GitHub Pages
  1. En tu repositorio, ve a Settings (esquina superior)
  2. En el menú izquierdo, haz clic en Pages
  3. En la sección "Source", selecciona:
    Deploy from a branch Branch: main (o master) Folder: / (root)
  4. Haz clic en Save
💡 GitHub puede tomar 1-2 minutos para procesar. Espera un poco y luego recarga la página.

Parte 5: Obtener Tu URL (Automático)

6
Encuentra tu URL de GitHub Pages
  1. Ve a Settings → Pages nuevamente
  2. Verás un mensaje verde que dice: "Your site is live at:"
  3. Copia la URL (será algo como):
    https://[tu-usuario].github.io/tribilin-web/
  4. ¡Esa es tu URL pública! Puedes compartirla con cualquiera
✅ ¡Tu sitio está en línea! La URL será permanente y podrás actualizarla en cualquier momento subiendo nuevos archivos.

🔧 Solución de Problemas

❌ "Page not found" (404)

Posibles soluciones:

  • Verifica que index.html esté en la raíz del repositorio
  • Espera 2-3 minutos después de subir los archivos
  • Intenta limpiar el caché del navegador (Ctrl+Shift+Delete)
❌ Los estilos no se ven (CSS no carga)

Solución:

  • Asegúrate que la carpeta css/ esté en el repositorio
  • Verifica que styles.css esté dentro de css/
  • Los paths en HTML deben ser relativos: css/styles.css ✓
❌ Los scripts no funcionan (JS no ejecuta)

Solución:

  • Verifica que la carpeta js/ esté en el repositorio
  • Verifica que script.js esté dentro de js/
  • Los paths deben ser relativos: js/script.js ✓

📝 Cómo Actualizar Tu Sitio

Si necesitas hacer cambios a tu sitio:

  1. Ve a tu repositorio en GitHub
  2. Haz clic en el archivo que quieres editar (ej: index.html)
  3. Haz clic en el ícono de lápiz (Edit)
  4. Realiza tus cambios
  5. En el campo "Commit message", describe el cambio
  6. Haz clic en Commit changes
  7. El sitio se actualizará automáticamente en 1-2 minutos
💡 También puedes usar Git desde tu computadora para cambios más complejos:
git clone https://github.com/[tu-usuario]/tribilin-web.git cd tribilin-web # Haz cambios a los archivos git add . git commit -m "Descripción del cambio" git push

🌐 Opción: Usar un Dominio Personalizado

Si tienes un dominio propio (ej: www.tribilin.pe):

  1. En tu registrador de dominio, configura los registros DNS
  2. En GitHub Settings → Pages, ve a "Custom domain"
  3. Ingresa tu dominio
  4. Sigue las instrucciones para configurar los registros DNS
💡 Esto es opcional. La URL de GitHub Pages es completamente funcional sin dominio personalizado.
